ptrace注入 frida

  1. 修改debuger 的注入

    ptrace描述

  2. https://langgithub.github.io/file/PTRACE函数代码分析.pdf
  3. https://github.com/langgithub/PtraceInject demo地址

注入流程

  1. 两种实现

    • 利用shellcode
    • 直接调用dlopen,dlsyn函数

  2. 流程

    • attch到远程进程 -> 保存寄存器环境 -> 远程调用mmap函数申请内存空间 -> 向远程进程内存空间写入加载模块名和调用函数
      -> 远程调用dlopen函数打开注入模块 -> 远程调用dlsym函数获取需要调用的函数地址 -> 远程调用被注入模块的函数
      -> 恢复寄存器环境 -> 从远程进程detach
      总结：远程进程暂停 -> ptrace函数调用,其他函数远程调用 -> 进程恢复

    获取模块基址: “/proc/pid/maps”
    相对偏移地址=本地进程函数地址-本地进程的模块基址
    获取远程进程函数地址=远程进程模块基址+相对偏移地址

  1. ptrace注入实现：
    1. attach 到远程进程
    2. 读取和写入寄存器值
    3. 远程进程的内存读取和写入数据
    4. 远程调用函数
      • 核心 ： 修改pc寄存器为需要执行的函数地址 lr寄存器致为0
      • mmap “system/lib/libc.so”
      • dlopen,dlsym,dlclose “/system/bin/linker”
    5. 恢复寄存器值
    6. detach进程
